# 内置能力表

为 JS/TS 标准库和 Node/Bun API 预声明【能力】的静态映射表。

当【调用解析】未能解析到项目内函数、且【外部声明文件】中也未找到匹配时，在此表中查找。

## 匹配方式

使用 ts-morph 的 `symbol.getFullyQualifiedName()` 作为主键匹配，而非裸函数名。

对于 PropertyAccess 调用（如 `obj.method()`），同时从 symbol 获取 receiverType（接收者类型名），构成 `{ReceiverType}.{method}` 的限定名。

### 匹配流程

1. 获取调用表达式的 symbol
2. 若 symbol 存在，取 `fullyQualifiedName`（如 `JSON.parse`、`Array.push`、`Console.log`）
3. 在内置能力表中按 fullyQualifiedName 精确匹配
4. 若 symbol 不存在或表中未命中，回退到裸函数名匹配（兼容无类型信息的场景）

### 限定名示例

| 调用代码 | fullyQualifiedName | 裸名 |
|----------|-------------------|------|
| `JSON.parse(s)` | `JSON.parse` | parse |
| `console.log(x)` | `Console.log` | log |
| `arr.push(1)` | `Array.push` | push |
| `Math.random()` | `Math.random` | random |
| `res.json()` | `Body.json` | json |
| `fetch(url)` | `fetch` | fetch |

自定义的同名函数（如用户写的 `parse()`）的 fullyQualifiedName 只有 `parse`，不会匹配到 `JSON.parse`。

### 表结构

从纯名称映射改为限定名映射：

```ts
// 旧：按裸名，有碰撞风险
{ "parse": ["Fallible"], "random": ["Impure"] }

// 新：按限定名，精确无碰撞
{
  "JSON.parse":    ["Fallible"],
  "Math.random":   ["Impure"],
  "Console.log":   ["IO"],
  "Array.push":    [],
  "Body.json":     ["Async", "Fallible"],
  "fetch":         ["IO", "Async", "Fallible"],
}
```

全局函数（如 `fetch`、`setTimeout`）的限定名就是函数名本身，无需特殊处理。

## 分类摘要

| 分类 | 限定名 | 能力 |
|------|--------|------|
| String 纯操作 | `String.trim`, `String.split`, `String.replace`... | 无（纯） |
| RegExp | `RegExp.test`, `RegExp.exec` | 无（纯） |
| Number | `Number.toFixed`, `Number.toString`... | 无（纯） |
| Math 纯操作 | `Math.abs`, `Math.floor`, `Math.ceil`... | 无（纯） |
| Math.random | `Math.random` | 【Impure】 |
| Date.now | `Date.now` | 【Impure】 |
| Array 读取类 | `Array.map`, `Array.filter`, `Array.reduce`... | 无（纯） |
| Array 变异类 | `Array.push`, `Array.pop`, `Array.splice`... | 无（纯）——局部变异不算 Mutable |
| JSON.parse | `JSON.parse` | 【Fallible】 |
| JSON.stringify | `JSON.stringify` | 无（纯） |
| Console | `Console.log`, `Console.warn`, `Console.error`... | 【IO】 |
| Timer | `setTimeout`, `setInterval` | 【Impure】 |
| fetch | `fetch` | 【IO】【Async】【Fallible】 |
| Response 方法 | `Body.json` → 【Async】【Fallible】；`Body.text` → 【Async】 |
| Node.js fs 同步 | `readFileSync`, `writeFileSync`, `mkdirSync` | 【IO】【Fallible】 |
| Node.js fs 异步 | `readFile`, `writeFile`, `stat`, `readdir`... | 【IO】【Async】【Fallible】 |
| Node.js path | `dirname`, `basename`, `resolve`, `relative`... | 无（纯） |
| process | `exit` → 【IO】；`cwd`, `argv` → 【Impure】 |
| Bun | `file` → 【IO】；`write` → 【IO】【Async】【Fallible】；`serve` → 【IO】 |

## 匹配中未命中

若调用的 fullyQualifiedName 和裸名都不在内置能力表中、也不在【外部声明文件】中、也未被【调用解析】解析到项目内函数，则：
- 该调用按全能力（5 个）处理
- 产生一条 【unregistered】 诊断
