# 输出格式

固定为 JSON，输出到 stdout。

## 完整输出（默认）

```json
{
  "diagnostics": [ ... ],
  "functions": [ ... ],
  "scores": { ... }
}
```

- **diagnostics**：【诊断】列表，每项含 kind, functionName, filePath(相对路径), line, message, missingCaps?, callee?
- **functions**：所有函数的 【函数得分】 信息，含 name, filePath, line, caps, isDeclared, 【加权语句数】, score
- **scores**：【能力负担分】、【类型松散度】、totalFunctions、totalPure、totalUndeclared、capScores（按能力分项）、looseByType（按信号类型分项）、topFunctions（前 10）、【文件得分】列表、【优化建议】（仅在传入 `--hint` 时包含）

所有 filePath 相对于 tsconfig.json 所在目录。

## --summary 输出

只包含 scores 部分，省略 diagnostics 和 functions：

```json
{
  "scores": { ... }
}
```

## 使用技巧

- 用 `jq` 提取特定字段：`capability-lint src/ | jq '.scores.totalCap'`
- 用 `rg` 搜索诊断：`capability-lint src/ | jq '.diagnostics[]' | rg missing_capability`
- 传入具体文件/目录缩小分析范围：`capability-lint src/api.ts`
