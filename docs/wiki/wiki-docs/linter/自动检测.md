# 自动检测

传播可阻断能力（【Fallible】、【Async】、【Mutable】）可从函数签名自动推断，无需手动声明。

## 规则

| 能力 | 触发条件 |
|------|----------|
| 【Fallible】 | 返回类型包含 null/undefined（含联合类型成员、泛型参数） |
| 【Async】 | 函数标记 async，或返回类型匹配 `Promise<` / `AsyncIterable<` / `AsyncGenerator<` / `AsyncIterableIterator<` |
| 【Mutable】 | 参数通过【参数可变性检测】判定为非 readonly 引用类型 |

满足条件时将能力加入 effectiveCaps，产生 【implicit_capability】 信息。

传播不可阻断能力（【IO】、【Impure】）无自动检测规则，只能手动声明。

阻断能力（【HandleFallible】、【HandleAsync】、【HandleMutable】）无自动检测规则，只能手动声明。

## 与阻断能力的关系

自动检测和阻断能力是两个独立维度：

- **自动检测**看的是函数**自身特征**（返回类型、参数类型）
- **阻断能力**管的是**调用链传播**（callee 的能力是否被 caller 处理）

两者互不干扰。即使函数声明了 HandleFallible，如果它自身返回 `T | null`，自动检测仍然标记 Fallible。
