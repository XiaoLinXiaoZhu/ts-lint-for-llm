# 编译器

将一个目录下的【fts文件】和【type.fts文件】合并为一个【编译产物】（index.ts）。

实现：`src/fts-compile.ts`，CLI 工具 `fts-compile`。

## 用法

```
fts-compile <dir> [<dir> ...]    编译指定目录
fts-compile --all                自动发现并编译所有含 .fts 的目录
fts-compile --all --watch        编译 + 监听文件变化自动重编译
```

## 编译步骤

1. 读取目录下所有 .fts / .type.fts 文件
2. 提取【index.fts】内容，原样输出到产物顶部
3. 剩余文件排序：类型优先 → 私有优先 → 字母序
4. 对每个文件按编译语义拼接（见【fts文件】【type.fts文件】）
5. 按【可见性】决定是否加 `export`
6. 首行 `/** @capability ... */` 原样保留（见【能力声明】）

不做 AST 解析或符号解析。【同目录共享作用域】消除了文件间 import 关系，编译器只做拼接。
