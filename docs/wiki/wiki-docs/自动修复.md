# 自动修复

通过 `--fix` 参数触发，基于【诊断】结果修改源文件中的 【JSDoc声明】。

## 执行时机

在分析（诊断产出）之后执行。若修改了文件，重新扫描+分析，输出修复后的结果。

## 修复规则与示例

### undeclared → 自动加空 @capability

```ts
// --fix 前
function helper(x: number): number { return x + 1; }

// --fix 后
/** @capability */
function helper(x: number): number { return x + 1; }
```

### missing_capability 缺失不可阻断能力 → 自动补声明

```ts
// --fix 前：process 调了 fetchUser(IO Fallible)，缺 IO
/** @capability */
function process(id: string) { fetchUser(id); }

// --fix 后：自动补了 IO（不可阻断，唯一选择）
// Fallible 是可阻断的，不自动补，保留诊断
/** @capability IO */
function process(id: string) { fetchUser(id); }
```

### missing_capability 缺失可阻断能力 → 不自动补

```ts
// --fix 不处理这种情况，保留诊断：
//   missing_capability: process 缺少 Fallible
// 等待人判断：
//   传播 → /** @capability IO Fallible */
//   阻断 → /** @capability IO HandleFallible */
```

### 多余声明 → 自动移除

```ts
// --fix 前：pureCalc 声明了 Mutable 但调用链中没遇到 Mutable
/** @capability Mutable */
function pureCalc(x: number): number { return add(x, 1); }

// --fix 后
/** @capability */
function pureCalc(x: number): number { return add(x, 1); }
```

例外：若函数有未解析调用（unresolvedCalls），不移除任何能力。

## 写入方式

在函数声明前 6 行内查找含 `@capability` 的行，替换能力列表。能力按固定顺序排列（IO, Impure, Fallible, Async, Mutable, HandleFallible, HandleAsync, HandleMutable）。从后往前按行号修改，避免偏移。

## --dry-run

配合 `--fix --dry-run` 预览变更，不实际写入文件。
