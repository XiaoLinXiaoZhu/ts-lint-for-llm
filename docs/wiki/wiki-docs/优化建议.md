# 优化建议

基于评分结果生成的提示。**不默认全量输出**——通过 `--hint` 参数按关键词筛选，只返回与当前关注点相关的建议，避免每次都输出相似内容导致信息被忽略。

## 用法

```
capability-lint --hint undeclared src/   # 只返回与未声明相关的建议
capability-lint --hint split src/        # 只返回与函数拆分相关的建议
capability-lint --summary src/           # 只返回分数和匹配的 tips
```

不传 `--hint` 时，输出中不包含 tips 字段。

## 可用的建议规则

| 条件 | 关键词 | 建议内容 |
|------|--------|----------|
| totalUndeclared > 0 | undeclared | 提示声明能力，说明【未声明函数】按最大惩罚计分 |
| 存在携带 ≥3 个能力的函数 | split | 提示拆分该函数，提取纯逻辑为独立纯函数。强调仅提取「能力更少」的代码才有效 |
| ≥3 个函数携带 ≥2 个能力 | refactor | 提示系统性重构（状态机模式、effect as data） |
| 纯函数占比 < 30%（且总函数数 > 3） | purity | 提示收窄接口，减少对外部能力的依赖 |
| totalCap > 0 且 totalLoose > 0 | priority | 提示优先降低【能力负担分】，再处理【类型松散度】 |
| totalCap > 0 且 totalLoose = 0 | priority | 提示集中精力降低【能力负担分】 |
| 存在同名函数出现在多个文件中 | duplicate | 提示提取到共享模块减少总能力面积 |
