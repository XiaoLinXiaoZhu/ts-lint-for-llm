# 调用解析

确定函数体内每个 CallExpression 调用的目标函数，分为"已解析"和"未解析"两类。

## 解析流程

### 1. 确定调用所属函数

从 CallExpression 沿 AST 向上查找最近的函数祖先节点，取其 `getStart()` + `filePath` 构成【函数ID】，直接在 functions 表中查到所属函数。无需名称匹配。

### 2. 解析调用目标

从 CallExpression 的表达式取 `symbol`（ts-morph 类型系统），再取 `symbol.getDeclarations()[0]`：

- 声明节点在项目内（非 node_modules）→ 取其 `getStart()` + `filePath` 构成目标【函数ID】，直接在 functions 表中查找。匹配成功即为"已解析"。
- 声明节点是 ImportSpecifier → 追踪到源模块的 export symbol，取其声明节点的 pos 构成 ID。
- 声明节点在 node_modules / typescript lib 中 → 进入"未解析"流程。
- 无 symbol 或无 declaration → 进入"未解析"流程。

### 3. 未解析的调用

解析失败时，记录：

- **fullyQualifiedName**：从 `symbol.getFullyQualifiedName()` 获取（如 `JSON.parse`、`Console.log`、`Array.push`）
- **裸名**：从调用表达式获取（Identifier → getText, PropertyAccess → getName），作为回退

存入 unresolvedCalls。

### 4. 未解析调用的能力查找

分析阶段按以下优先级：
1. 【外部声明文件】（按裸名匹配）
2. 【内置能力表】（优先按 fullyQualifiedName 精确匹配，回退裸名）
3. 均未命中 → 按全能力处理，产生 【unregistered】 诊断
