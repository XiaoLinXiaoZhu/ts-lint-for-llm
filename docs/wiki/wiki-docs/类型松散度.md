# 类型松散度

衡量项目中类型不精确程度的总分。越低越好。独立于【能力负担分】。

## 检测信号与权重

| 信号 | 权重 | 说明 |
|------|------|------|
| `any` | 10 | 类型标注中的 any |
| `as any` | 8 | as any 断言 |
| `Record<string, any>` | 8 | 值类型为 any 的 Record |
| `Object` | 8 | 大写 Object 类型引用 |
| `Function` | 6 | Function 类型引用 |
| `object` | 5 | 小写 object 关键字 |
| `Record<string, unknown>` | 5 | 值类型为 unknown 的 Record |
| `{}` | 5 | 空类型字面量 |
| `unknown` | 3 | unknown 关键字 |
| `x!` | 2 | 非空断言 |
| `boolean` 参数 | 2 | 函数参数类型为 boolean |
| 可选属性 `?` | 1 | interface 或 type 中带 `?` 的属性签名（PropertySignature with questionToken） |
| `@ts-ignore` | 10 | 注释中的 @ts-ignore |
| `@ts-expect-error` | 10 | 注释中的 @ts-expect-error |

## 计算

%% [类型松散度] = 所有信号的权重之和 %%

按信号类型汇总提供 byType 分项（count 和 penalty）。

## 扫描范围

项目中所有源文件（排除 node_modules 和 .cap.ts）。基于 ts-morph AST 遍历 + 逐行文本扫描（用于 @ts-ignore/@ts-expect-error）。
