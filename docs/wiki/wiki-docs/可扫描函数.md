# 可扫描函数

以下 5 种形式被识别为函数，纳入扫描并分配【函数ID】：

| # | 形式 | 示例 |
|---|------|------|
| 1 | 顶层 function 声明 | `function foo() {}` |
| 2 | 变量绑定的箭头函数或函数表达式 | `const foo = () => {}` |
| 3 | 对象字面量中的方法声明 | `{ method() {} }` |
| 4 | 对象字面量中的属性赋值为箭头函数/函数表达式 | `{ getUser: (id) => ... }` |
| 5 | class 方法 | `class Foo { bar() {} }` |

不扫描：
- node_modules 内的文件
- `.cap.ts` 后缀的文件（这些是【外部声明文件】）
- 匿名 function 声明（无名称）

每个扫描到的函数记录以下属性：

| 属性 | 说明 |
|------|------|
| id | 【函数ID】 |
| name | 函数名 |
| filePath | 所在文件绝对路径 |
| line | 起始行号 |
| declaredCaps | 【能力声明】解析结果（含传播能力和阻断能力） |
| isDeclared | 是否有显式声明 |
| returnsAsync | 返回类型是否含 Promise/AsyncIterable（用于【Async】【自动检测】） |
| returnsNullable | 返回类型是否含 null/undefined（用于【Fallible】【自动检测】） |
| mutableParams | 通过【参数可变性检测】判定的非 readonly 引用类型参数名列表 |
| resolvedCalls | 函数体内已解析的调用列表（target=【函数ID】, line） |
| unresolvedCalls | 函数体内未解析的调用列表（target=函数名, qualifiedName=fullyQualifiedName, line） |
| weightedStatements | 【加权语句数】 |
| statementCount | 原始语句数 |
