# 能力声明

每个函数必须声明自己携带哪些【能力】。三种方式，优先级从高到低：

## 1. 【后缀命名】

函数名用 `_` 分隔，其中匹配能力名的部分即为声明：`fetchUser_IO_Async`。

## 2. 【JSDoc声明】

```ts
/** @capability IO HandleFallible */
function getUsername() {}
```

空 `@capability` = 纯函数。8 个能力名均可列出。

## 3. 【未声明函数】

无后缀也无 @capability 的函数，视为携带全部 5 个传播能力，按最大惩罚计分。

---

声明结果产出：
- **declaredCaps**：声明的能力集合（含传播能力和阻断能力）

函数的**实际能力**（effectiveCaps）= declaredCaps + 【自动检测】注入的传播能力（Async、Fallible、Mutable）。自动检测基于函数自身特征，不受阻断能力影响。

向调用方传播的能力（propagatedCaps）= effectiveCaps - 阻断能力 - 被阻断的传播能力。调用链检查用 propagatedCaps。
