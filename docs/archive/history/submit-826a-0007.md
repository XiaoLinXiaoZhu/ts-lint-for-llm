# ✅ 任务完成

## 总结

已实现能力健康报告工具 `prototype/src/report.ts`，对项目所有函数做三级分类并生成可视化报告。

三级分类：
- ✅ **好函数**：已声明且能力 ⊆ {Async, Blocking, Fallible, Mutable}（方便单测）
- 🔶 **已声明**：标注了能力但含 IO/Impure/Unsafe 等（有副作用但透明）
- ❌ **未声明**：无任何标注 = 坏函数

同时根据你的三点反馈更新了整个方案：

1. **未标注 = 全能力坏函数**（而非纯函数）。ESLint 规则对未标注函数报 `undeclared` 警告，逼迫开发者声明能力。已验证通过。

2. **完整英文单词**：`_IO_Blocking_Fallible` 替代 `_IBE`。函数名后缀是首选标注方式（调用处可见），JSDoc `@capability` 是不方便改名时的兜底。

3. **外部模块适配**：`.caps.ts` 声明文件（数据 SSOT）→ `codegen.ts` 自动生成 → `adapters/xxx.ts`（re-export 带能力后缀）。已跑通 `node:fs` 的完整链路，生成了 19 个函数的适配层。

改造前后对比效果：0% 健康度 → 100% 健康度，CI 门禁可配置未声明坏函数的最大占比阈值（默认 20%）。
